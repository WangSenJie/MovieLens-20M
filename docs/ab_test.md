# 推荐系统 A/B Test 思路

这个项目的线上实验目标不是证明“模型更复杂”，而是证明“用户体验和业务指标更好”。

## 1. 实验目标

比较两类方案：

- Control：单阶段推荐，例如 `svd` 或 `lightfm`
- Treatment：两阶段推荐，先用 `content + itemcf + popularity` 召回，再用 `svd / lightfm` 重排

## 2. 核心假设

- 两阶段架构会提高点击率和播放转化率
- 两阶段架构能在不显著恶化延迟的前提下，带来更好的长尾曝光

## 3. 指标设计

北极星指标：

- CTR
- Watch Rate
- Favorite / Like Rate

护栏指标：

- P95 延迟
- Bounce Rate
- Complaint Rate
- Long-tail Coverage

## 4. 分层策略

建议至少分以下几层观察：

- 新用户
- 活跃用户
- 重度观影用户

## 5. 实验周期

- 建议 7-14 天
- 如果日活较低，优先保证样本量足够，再决定是否提前结束

## 6. 上线建议

- 先灰度 5%
- 验证延迟和接口稳定性
- 再逐步扩到 20%、50%、100%
